História da WEB

Voltar

A primeira geração de computadores, foi em 1944 em Harvard, um computador basicamente de cálculo chamado Mark I. Em 1946 foi a segunda geração marcada pelo computador alemão chamado Colossus, e pelo computador Americano chamado Eniac. Entre 1959 a 1970 foi a terceira geração, quando surgiu os transistores(componentes menores) que permitiu diminuir o tamanho físico dos computadoes. Ainda na terceira geração, entre 1965 a 1970, teve o surgimento dos circuitos integrados. Já a quarta geração começou em 1971 e continuamos nessa geração até os dias atuais. Algumas pessoas dizem que a quinta geração está próxima com o avanço da inteligência artificial(IA) e os computadores quânticos.

Em 1958, surgiu a primeira comunicação intercontinental, atravéz de um telégrafo. Vinton Cerf e Robert Kahn criaram a primeira conexão TCP/IP feita em 29/10/1969. O primeiro navegador surgiu em 1991, o World Wide Web(WWW) uniu os hipertextos com o TCP/IP seu criador se chama Tim Berners Lee.

Outros grandes nomes da tecnologia são Gracy Murray Hopper, a primeira programadora do Mark I; Licklider "plantou" a semente da comunicação entre computadores; Robert Kahn fez a primeira apresentação da ARPANET e enviou o primeiro e-mail; Marc Andressen reinventor do navegador NetScape; Bill Gates fundador da Micrisoft e criador do internet Explore; Steve Jobs fundador da Apple e popularizou os smathphones.

O que são Clients

Voltar

Clients são os consumidores de informação da internet, enquanto os servers, são os disponibilizadores de informações(arquivos estáticos) para os clients.Os arquivos estáticos são, html, css, JavaScript, imagens,audios e outros.

É de grande importancia para os clients o design responsivo ou responsividade, que são diferentes resoluções para diferentes dispositivos, e deve ser considerado na criação de um website. Os navegadores(browsers) são programas criados para abrir/executar os arquivos e seguem o padrão W3C. já as aplicações WEB são soluções criadas que possuem a internet como meio de comunicação entre client x server, não sendo necessária a sua instalação.

O que são Servers

Voltar

Como já dito anteriormente, os servers são os disponibilizadores de informações(arquivos estáticos) para os clients.Os arquivos estáticos são, html, css, JavaScript, imagens, audios e outros. São mais robustos e podem ser de arquivo, seguraça, streaming, e-mail, web e outros. Pode ser um servidor de rede local(LAN) ou não. existem os conjuntos de servidores, que são os DATACENTER, gigantescos.

Os servers são divididos em hardware e software. Os Hardware são de armazenamento(Disco rígido), memória e processadores, já os Software são os sistemas operacionais, monitoramento e servidores WEB. Um servidor vai ter um sistema operacional e um servidor WEB.

Existem vários tipos de servidores, os pricipais são: Proxy, que bloqueia algum tipo de site indezejado; Firewal que bloqueia alguns acessos externos; DataBase Server, é o servidor que acessa o banco de dados; DNS que redireciona um nome para um dominio(IP).

Linguagens de programação

Voltar

A linguagem de programação é uma linguagem escrita e formal que especifíca um conjunto de instrucões e regras para gerar um programa.

As linguagens de programação de server-side são: PHP, Ruby, C#, JavaScript, Java. e a linguagem de progarmação client-side é o JavaScript. O V8 é o "interpretador" do JavaScript.

O HTML não é considerado uma linguagem de programação e sim um arquivo de marcação.

HTML na prática

Voltar

Tags

As tags são formadas por uma estrutura própria, iniciam com o sinal “menor que”, em seguida vem o nome daquele elemento e por fim, o sinal “maior que”. Podem ser dispostas em tags que precisam de fechamento e tags que fecham sozinhas (self-closing). O fechamento de uma tag será definido com uma barra (/), sendo que no caso das tags de auto fechamento, não há necessidade da presença desse caractere.

Atributos das tags

Atributos HTML são palavras especiais usadas dentro da tag de abertura para controlar o comportamento do elemento. Os atributos HTML são um modificador de um tipo de elemento HTML. Com eles podemos identificar melhor um elemento, informar qual arquivo aquela tag deve utilizar, indicar o tipo de um campo de texto, etc…
